Global object setup

If xduce is not in a CommonJS or AMD environment, then it exposes an object called xduce on the global object. In a browser, this is the window object. If you want to use xduce in a browser directly, this is the setup to follow.

Installation

Download the latest package from the GitHub releases page. The file you want is in the dist directory; you can use either xduce.js (unminified) or xduce.min.js (minified). The other files in that directory are ES2015 and should only be used if you're abandoning Internet Explorer users.

It is suggested that you use the unminified version of the library in development environments (for easier debugging) and the minified version in production environments (for smaller size and faster loading).

Just put the file of your choice in a location that matches the location indicated by the src attibute in your <script> tag.

Usage

All of xduce's functionality is available on an object named xduce that is attached directly to window and can therefore be called without qualification (just as xduce rather than window.xduce). You have access to it directly after loading the xduce file. 


    <!-- use the path to your xduce.js file here -->
    <script src="xduce.js"></script>
    <script>
      var map = xduce.transducers.map;

      map([1, 2, 3, 4, 5], function(x) { return x + 1; });
      // -> [2, 3, 4, 5, 6]
    </script>
