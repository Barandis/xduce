AMD setup

AMD is the module system used by RequireJS and has been a popular way to use server-side modules in the browser for many years. If you intend to use xduce along with RequireJS, then this how you do it.

Installation

Download the latest package from the GitHub releases page. The file you want is in the dist directory; you can use either xduce.js (unminified) or xduce.min.js (minified). The other files in that directory are ES2015 and should only be used if you're abandoning Internet Explorer users.

It is suggested that you use the unminified version of the library in development environments (for easier debugging) and the minified version in production environments (for smaller size and faster loading).

You can use the file of your choice as an AMD module as-is; just put it a place that matches the location indicated by your requirejs or define call.

Usage

All of xduce's functionality is available from the base object that is required from the module. Just require the object and use it the way you would use any other object. (As shown below, the transducers are actually available from the transducers object that is a property of the main object; see the API documentation for details about where to find what.) 


    // use the path to your xduce.js file here
    requirejs(['./xduce'], function(xduce) {
      var map = xduce.transducers.map;

      map([1, 2, 3, 4, 5], function(x) { return x + 1; });
      // -> [2, 3, 4, 5, 6]
    });

If you're using a transpiler (like Babel) to turn ES2015 modules into AMD modules, you can instead use import statements. 


    import xduce from './xduce';
    const map = xduce.transducers.map;

    map([1, 2, 3, 4, 5], x => x + 1);
    // -> [2, 3, 4, 5, 6]
