Xduce is a JavaScript library that implements transducers in the spirit of Clojure transducers.

A transducer is a generalization of all of the map, filter, and similar functions that functional programming promotes. If there is a problem with these functions, it's that they can only be instructed on how to transform values in a custom way, and that's only one part of a collection-related function. They cannot be instructed on how to break a collection into its component parts for transformation (iteration), and they cannot be instructed on how to take the transformed values and put them back together into a new collection (reduction). So while you can use Array.prototype.map to turn any type of array element into any other type of array element, they have to be array elements. You can't use this function on anything but an array, and you can't have it return anything other than an array.

Xduce seeks to improve this by providing an engine that allows customization of all three parts of the process: iteration, transformation, and reduction. This means the same map function can be used on arrays, strings, objects, iterators, streams, channels, immutable lists, or anything else.

Installation

In node.js, or for use with Browserify, Webpack, etc., you can install the npm package.


          npm install xduce

or, if you use yarn


          yarn add xduce

For the browser, you can visit the releases page to download the latest archive. Inside there is a dist folder that contains xduce.js and its minified version, xduce.min.js. These use transpiled ES5 code; if you prefer ES2015, you can instead use either xduce.es.js or xduce.es.min.js. Drop the file of your choice into your project and reference it as normal in your HTML file.

Usage

Xduce provides all of its functionality on a single exported object.

In node.js, Browserify, Webpack, etc., you can access this through require or ES2015 import. 


          import xduce from 'xduce';
          const { map } = xduce.transducers;

          const array = map([1, 2, 3, 4, 5], x => x + 1);
          // array = [2, 3, 4, 5, 6]
          const string = map('hello', x => x + x);
          // string = 'hheelllloo'

If you're using require.js, you can use requirejs.


          requirejs(['xduce'], xduce => {
            const { map } = xduce.transducers;

            const array = map([1, 2, 3, 4, 5], x => x + 1);
            // array = [2, 3, 4, 5, 6]
            const string = map('hello', x => x + x);
            // string = 'hheelllloo'
          });

Finally, if you're using directly in an HTML page, you can use the global xduce object.


          <script src="js/xduce.js"></script>
          <script>
            const { map } = xduce.transducers;

            const array = map([1, 2, 3, 4, 5], x => x + 1);
            // array = [2, 3, 4, 5, 6]
            const string = map('hello', x => x + x);
            // string = 'hheelllloo'
          </script>

API documentation

Every public API function is documented the API documentation. Most of the functions have examples of their usage. In the future I plan to have more documentation on further usage of some of these functions, general concepts, and writing your own transducers.

Inspiration

Transducers were introduced to me after they were brought into Clojure, particularly in conjunction with CSP, which I have also written a library for. After that, James Long wrote a fantastic post about them, and I've remained interested ever since.
